# open-claw-agent-docker-setup

A minimal Docker / VS Code Dev Container recipe that gives you a clean Ubuntu
24.04 environment with [OpenClaw](https://github.com/openclaw/openclaw) and
[Claude Code](https://docs.anthropic.com/en/docs/claude-code) pre-installed and
ready to run.

It exists because the default OpenClaw install flow assumes a user `systemd`
unit, which doesn't exist inside a container, and the `openclaw dashboard`
helper relies on a clipboard that also doesn't exist in headless containers.
This repo bakes in the workarounds for both.

> Unaffiliated with Anthropic or the OpenClaw project. The image fetches both
> CLIs from their upstream sources at build time.

## What's in the box

| File | Purpose |
| --- | --- |
| `Dockerfile` | Ubuntu 24.04 + Node.js 22 (LTS) + `openclaw` + `claude`, running as the stock `ubuntu` user (UID 1000) with passwordless `sudo`. |
| `.devcontainer/devcontainer.json` | VS Code / Cursor Dev Container config. Builds the image, forwards port `18789` (OpenClaw gateway), mounts persistent volumes for OpenClaw and Claude state, and prints first-run hints. |
| `scripts/print-openclaw-dashboard-url.js` | Reads `~/.openclaw/openclaw.json` and prints `http://127.0.0.1:<port>/#token=<token>` so you can open the Control UI when `openclaw dashboard` can't reach a clipboard. |
| `.dockerignore` | Keeps `.git`, `.devcontainer`, `node_modules`, etc. out of the build context. |

## Requirements

- Docker (or Podman with Docker compatibility)
- Optional: VS Code or Cursor with the **Dev Containers** extension

OpenClaw requires Node.js >= 22.14, so the image pins the Node 22 LTS line from
NodeSource.

## Quickstart — VS Code / Cursor Dev Container

1. Clone this repo and open the folder in VS Code or Cursor.
2. When prompted, **Reopen in Container** (or run "Dev Containers: Rebuild and
   Reopen in Container" from the command palette).
3. Once the container is up, in the integrated terminal:

   ```bash
   openclaw onboard          # configure providers / API keys (interactive)
   openclaw gateway run      # start the WebSocket gateway in the foreground
   ```

4. In a second terminal, get a clickable Control UI URL with the auth token
   embedded:

   ```bash
   node scripts/print-openclaw-dashboard-url.js
   ```

   Open the printed URL in your browser. Port `18789` is auto-forwarded.

## Persistent state

The Dev Container automatically attaches two named Docker volumes so your
config survives "Rebuild Container":

| Volume | Mounted at | Holds |
| --- | --- | --- |
| `openclaw-state` | `/home/ubuntu/.openclaw` | Gateway auth token, provider/API key config, session state. |
| `claude-state` | `/home/ubuntu/.claude` | Claude Code login, `settings.json`, MCP config, conversation history. |

Docker creates these on first launch — no manual setup required. You can also
pre-create them or inspect them anytime:

```bash
docker volume create openclaw-state
docker volume create claude-state
docker volume ls | grep -E "openclaw-state|claude-state"
docker volume inspect openclaw-state
```

Back up a volume to a tarball:

```bash
docker run --rm -v openclaw-state:/data -v "$PWD":/backup ubuntu \
  tar czf /backup/openclaw-state.tgz -C /data .
```

If you only authenticate Claude via `ANTHROPIC_API_KEY` (env var) and never
run `claude login`, the `claude-state` volume is harmless but unused — you can
remove it from `.devcontainer/devcontainer.json` if you prefer.

## Quickstart — plain Docker

```bash
docker build -t openclaw-dev .

docker run --rm -it \
  -p 18789:18789 \
  -e ANTHROPIC_API_KEY \
  -v "$PWD":/workspace \
  -v openclaw-state:/home/ubuntu/.openclaw \
  -v claude-state:/home/ubuntu/.claude \
  openclaw-dev bash
```

Then inside the container:

```bash
openclaw onboard
openclaw gateway run
# in another `docker exec` session:
node /workspace/scripts/print-openclaw-dashboard-url.js
```

## Why these workarounds?

- **No `systemd` in the container.** OpenClaw's docs steer you toward
  `openclaw gateway install` / `restart`, which manage a user `systemd` unit.
  In Docker there's no user `systemd`, so you need to run the gateway in the
  foreground with `openclaw gateway run` and let Docker / your IDE manage the
  process lifecycle.
- **No clipboard for `openclaw dashboard`.** That command tries to copy the
  Control UI URL (with the embedded auth token) to your clipboard. In a
  headless container that fails. `scripts/print-openclaw-dashboard-url.js`
  reads the same config file and just prints the URL to stdout instead.
- **Passwordless `sudo` for the `ubuntu` user.** Convenient for ad-hoc
  `apt-get install` while developing. **Fine for a dev container, do not copy
  this pattern into a production image.**

## Security notes

- No secrets are baked into the image. The OpenClaw auth token is generated by
  `openclaw onboard` at runtime and stored in `~/.openclaw/openclaw.json`
  inside the container — keep that file out of any image layers and out of git
  (this repo's `.gitignore` covers it).
- The image installs `openclaw@latest` and the latest Claude Code release at
  build time. For reproducible builds, pin specific versions in the
  `Dockerfile`.
- Port `18789` is bound to `127.0.0.1` only when forwarded by the Dev
  Container; if you publish it with `docker run -p`, make sure you understand
  who can reach it on your network.
